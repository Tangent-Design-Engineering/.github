# Tangent Design Engineering Organization Community Guidelines

This repository contains the default community health files for Tangent Design Engineering.

These files define how contributors should interact with repositories across the organization and provide templates for issue and Pull Request creation.

## Purpose

The files in this repository standardize how we:

- Report bugs
- Request new features
- Submit pull requests
- Maintain code quality and documentation

These defaults are automatically applied to repositories in this organization that do not define their own versions of these files.

---

# Included Community Health Files

This repository includes the following files:

| File | Purpose |
|-----|--------|
| `PULL_REQUEST_TEMPLATE.md` | Standard template used when creating pull requests |
| `.github/ISSUE_TEMPLATE/*` | Issue templates for bug reports, feature requests, and documentation updates |
| `CONTRIBUTING.md` | Guidelines for contributing to repositories |

---

# Pull Request Guidelines

All pull requests should:

- Use naming matching the feature branch ("lparker/1-initial-commit")
- Use closing keywords
- Reference related issues when applicable
- Describe the purpose/summary of the changes
- Include testing steps
- Follow project coding standards
- Update documentation if necessary

The pull request template will guide contributors through these requirements.

---

# Issue Templates

Issue templates help ensure that bug reports and feature requests contain the information required to address them efficiently.

Templates include:

- **Bug report**
- **Feature request**
- **Development Tasks**
- **Documentation issue**
- **Question / support request**
